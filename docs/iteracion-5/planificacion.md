# Iteración 5: Dashboard y Métricas

## Planificación

### Selección de Historias de Usuario
El objetivo de esta iteración es proporcionar a los docentes una visión general del progreso del proyecto y de los estudiantes. La historia de usuario principal es:

- **HU 8 (Docente):** "Como docente, quiero acceder a métricas y reportes sobre el progreso de los sprints, la contribución individual de los miembros del equipo y el desempeño general del proyecto para evaluar objetivamente el trabajo realizado."

### Estimación de Tiempos
El tiempo estimado para esta iteración es de **2 semanas**.

- **Semana 1:** Diseño e implementación del Dashboard para docentes.
- **Semana 2:** Desarrollo del sistema de reportes y pruebas.

### Definición de Tareas Específicas
- **T5.1:** Diseñar la interfaz del Dashboard del docente.
- **T5.2:** Desarrollar los componentes de la interfaz de usuario para visualizar las métricas clave (progreso del Sprint, contribución individual, etc.).
- **T5.3:** Implementar las consultas a la base de datos para obtener las métricas.
- **T5.4:** Desarrollar la funcionalidad para generar reportes en formato PDF.
- **T5.5:** Integrar el Dashboard con los módulos existentes.
- **T5.6:** Realizar pruebas de rendimiento del Dashboard.

### Entregables
- **Plan de Iteración:** Documento actualizado.
- **Fichas de Tareas:** Nuevas tarjetas en el tablero de gestión de proyectos.
