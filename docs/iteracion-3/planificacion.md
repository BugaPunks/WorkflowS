# Iteración 3: Seguimiento y Kanban

## Planificación

### Selección de Historias de Usuario
El objetivo de esta iteración es implementar el tablero Kanban y las herramientas de seguimiento visual. La historia de usuario principal es:

- **HU 6 (Miembro del equipo):** "Como miembro del equipo, quiero visualizar las tareas en un tablero Kanban y poder arrastrarlas entre columnas para actualizar su estado de manera intuitiva y seguir el progreso del proyecto."

### Estimación de Tiempos
El tiempo estimado para esta iteración es de **2 semanas**.

- **Semana 1:** Implementación del tablero Kanban con funcionalidad de arrastrar y soltar.
- **Semana 2:** Desarrollo de los gráficos de seguimiento y pruebas.

### Definición de Tareas Específicas
- **T3.1:** Implementar el tablero Kanban utilizando una librería de arrastrar y soltar (drag-and-drop) como `react-beautiful-dnd`.
- **T3.2:** Conectar el tablero Kanban con la API para actualizar el estado de las tareas en la base de datos.
- **T3.3:** Diseñar y desarrollar los gráficos de avance del proyecto (Burndown Chart).
- **T3.4:** Integrar una librería de gráficos como `Chart.js` o `Recharts` para visualizar las métricas.
- **T3.5:** Escribir pruebas para la funcionalidad del tablero Kanban.

### Entregables
- **Plan de Iteración:** Documento actualizado.
- **Fichas de Tareas:** Nuevas tarjetas en el tablero de gestión de proyectos.
