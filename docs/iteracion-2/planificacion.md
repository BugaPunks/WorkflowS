# Iteración 2: Sprints y Backlogs

## Planificación

### Selección de Historias de Usuario
Para esta iteración, el enfoque es la gestión de Sprints y el Product Backlog. Las historias de usuario seleccionadas son:

- **HU 5 (Team Developer):** "Como Team Developer, quiero crear, modificar y actualizar el estado de las tareas asociadas a las historias de usuario para gestionar mi trabajo diario y mantener informado al equipo sobre mi progreso."
- **HU 6 (Miembro del equipo):** "Como miembro del equipo, quiero visualizar las tareas en un tablero Kanban y poder arrastrarlas entre columnas para actualizar su estado de manera intuitiva y seguir el progreso del proyecto."

### Estimación de Tiempos
El tiempo estimado para esta iteración es de **3 semanas**.

- **Semana 1:** Diseño y modelado de datos para Sprints y Backlogs.
- **Semana 2:** Implementación del CRUD de Sprints y tareas.
- **Semana 3:** Desarrollo del tablero Kanban y pruebas de integración.

### Definición de Tareas Específicas
- **T2.1:** Extender el esquema de la base de datos para incluir Sprints, Historias de Usuario y Tareas.
- **T2.2:** Implementar el CRUD para Sprints dentro de un proyecto.
- **T2.3:** Desarrollar el CRUD para Historias de Usuario (Product Backlog).
- **T2.4:** Implementar el CRUD para Tareas, asociadas a las historias de usuario.
- **T2.5:** Diseñar la interfaz de usuario para la gestión de Sprints y el Backlog.
- **T2.6:** Desarrollar la funcionalidad para asignar tareas a los miembros del equipo.
- **T2.7:** Escribir pruebas para la lógica de negocio de Sprints y Backlogs.

### Entregables
- **Plan de Iteración:** Documento actualizado con las nuevas historias, estimaciones y tareas.
- **Fichas de Tareas:** Nuevas tarjetas en el tablero de gestión de proyectos.
