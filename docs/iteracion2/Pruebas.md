# Pruebas de la Iteración 2: Sprints y Backlogs

## Pruebas Unitarias

- **Creación de Sprints:** Se crearán pruebas unitarias para verificar que la creación de sprints a través de la API funciona correctamente.
- **Asignación de Historias de Usuario:** Se probará que la asignación de historias de usuario a los sprints funciona como se espera.

## Pruebas de Integración

- **Ciclo de Vida del Sprint:** Se realizarán pruebas de integración para asegurar que se puede crear un sprint, asignarle historias de usuario, y luego actualizarlo o eliminarlo.

## Pruebas de Interfaz de Usuario

- **Gestión de Sprints:** Se probará la interfaz de gestión de sprints para verificar que los docentes y scrum masters pueden crear, actualizar y eliminar sprints.
- **Backlog del Sprint:** Se probará la interfaz del backlog del sprint para asegurar que los miembros del equipo pueden ver las historias de usuario asignadas.
