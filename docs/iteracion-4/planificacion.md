# Iteración 4: Evaluación y Calificación

## Planificación

### Selección de Historias de Usuario
El enfoque de esta iteración es el sistema de evaluación y retroalimentación. La historia de usuario principal es:

- **HU 7 (Docente):** "Como docente, quiero evaluar los entregables de los estudiantes con criterios predefinidos y proporcionar retroalimentación detallada para ayudarles a mejorar y asignar calificaciones justas."

### Estimación de Tiempos
El tiempo estimado para esta iteración es de **3 semanas**.

- **Semana 1:** Diseño del sistema de evaluación y modelado de datos.
- **Semana 2:** Implementación de los formularios de evaluación y retroalimentación.
- **Semana 3:** Desarrollo del cálculo automático de calificaciones y pruebas.

### Definición de Tareas Específicas
- **T4.1:** Diseñar y extender el esquema de la base de datos para incluir Evaluaciones, Calificaciones y Retroalimentación.
- **T4.2:** Crear la interfaz de usuario para que los docentes puedan definir criterios de evaluación.
- **T4.3:** Implementar los formularios para que los docentes puedan calificar los entregables y escribir retroalimentación.
- **T4.4:** Desarrollar la lógica para calcular automáticamente las calificaciones basadas en los criterios de evaluación.
- **T4.5:** Implementar un sistema de notificaciones para informar a los estudiantes sobre sus calificaciones y retroalimentación.
- **T4.6:** Escribir pruebas unitarias para el cálculo de calificaciones.

### Entregables
- **Plan de Iteración:** Documento actualizado.
- **Fichas de Tareas:** Nuevas tarjetas en el tablero de gestión de proyectos.
