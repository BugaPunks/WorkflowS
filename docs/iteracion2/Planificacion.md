# Planificación de la Iteración 2: Sprints y Backlogs

## Historias de Usuario

- Como docente, quiero poder crear y gestionar sprints dentro de un proyecto.
- Como scrum master, quiero poder asignar historias de usuario a los sprints.
- Como miembro del equipo, quiero poder ver el backlog del sprint y las historias de usuario asignadas.

## Tareas Específicas

- Diseñar el modelo de datos para sprints e historias de usuario.
- Implementar la API para la creación, actualización y eliminación de sprints.
- Implementar la API para la creación y gestión de historias de usuario.
- Implementar la API para asignar historias de usuario a los sprints.
- Crear la interfaz de usuario para la gestión de sprints y el backlog.
