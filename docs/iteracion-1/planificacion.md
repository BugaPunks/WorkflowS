# Iteración 1: Gestión de Equipos y Roles

## Planificación

### Selección de Historias de Usuario
Para esta iteración, se seleccionaron las siguientes historias de usuario clave, enfocadas en establecer las bases del sistema de gestión de proyectos:

- **HU 2 (Docente):** "Como docente, quiero crear proyectos, definir sus fechas de inicio y fin, y asignar estudiantes con roles específicos (Scrum Master, Product Owner, Team Developer) para organizar el trabajo colaborativo de manera efectiva."
- **HU 3 (Product Owner):** "Como Product Owner, quiero crear, modificar y priorizar historias de usuario, así como definir criterios de aceptación para cada una, con el fin de establecer claramente los requisitos del proyecto."

### Estimación de Tiempos
El tiempo estimado para completar esta iteración es de **2 semanas**.

- **Semana 1:** Diseño de la arquitectura, modelado de la base de datos e implementación del CRUD de usuarios y proyectos.
- **Semana 2:** Implementación de la asignación de roles, gestión de historias de usuario y pruebas unitarias.

### Definición de Tareas Específicas
Las tareas concretas a desarrollar son:
- **T1.1:** Configurar el entorno de desarrollo con Next.js, Prisma y Tailwind CSS.
- **T1.2:** Diseñar el esquema de la base de datos para usuarios, proyectos y roles.
- **T1.3:** Implementar el CRUD (Crear, Leer, Actualizar, Eliminar) para usuarios.
- **T1.4:** Desarrollar el CRUD para proyectos.
- **T1.5:** Implementar la lógica para asignar roles a los usuarios dentro de un proyecto.
- **T1.6:** Crear la interfaz de usuario para la gestión de equipos y roles.
- **T1.7:** Escribir pruebas unitarias para la validación de roles y permisos.

### Entregables
- **Plan de Iteración:** Documento que detalla las historias de usuario, estimaciones y tareas.
- **Fichas de Tareas:** Tarjetas en el tablero de gestión de proyectos (Trello/Jira) para cada tarea específica.
